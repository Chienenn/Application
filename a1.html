WeHelp Bootcamp

    個人簡介

    我是lily，畢業於長庚科大護理系，畢業後在林口長庚醫院擔任護理師至今三年半，工作性質的關係，個性方面謹慎小心，有獨立的判斷力也具有團隊合作的能力，是個意志堅定，只要有想做的事就會堅持下去的人。萌生離職的想法是從這幾個月開始，政府政策及醫院環境令人灰心，於是開始學習其他新的事物。剛好有相關產業的朋友推薦我從Python入門，當作學習新技能。這兩個月開始利用下班的時間學習，可以為了找一個bug坐在電腦前一下午，從Hello World到做出簡易網頁，每天學習到不同的新知另我感到愉快。與人類不同，寫程式時可以即時反饋所寫的正確性，這種感覺給我帶來踏實感，對此越來越有興趣，也持續學習著。平時常看彭彭老師的教學，發現Bootcamp的招生訊息，決定報名參加。

    為了成為軟體工程師，曾做過什麼努力？請分享具體專案作品

    目前是利用下班後及放假的時間學習，從youtube及udemy尋找課程分別是
    • 彭彭老師的python入門課程
    • 彭彭老師的Html CSS JavaScript 前端入門課程
    • The Complete JavaScript Course 2023
    • Advanced CSS and Sass課程
    • HTML5、CSS、JavaScript、jQuery、jQuery UI、Ajax、RWD網頁程式設計
    影片所教的程式都會自己跟著輸入一遍並思考其中的含意，遇到問題會先從Google、Stack Overflow..等網頁查詢，重點或筆記利用One note整理，也從教學裡學習到的做了簡單的小遊戲。

    若參與這個計畫，會如何安排學習時間

    了解自己的最終目標是轉職成功，非本科轉職的關係需要比別人花上更多的時間學習，若能參加計畫會是無業的狀態下去準備，每天必須花10小時以上學習至自己滿意的進度為止，訂定每週可量化的計畫，寫下每日進度，並適時作出調整，掌控自己的學習進度。若確定參與，再正式開始前，把老師列出的每週課程大綱裡的相關技術當作目標提早學習，會抱著沒有退路的心情去準備。

    想要加入的軟體公司？為什麼想加入該公司？

    想加入的公司是KKBOX，自己是個多年的KKBOX用戶，很喜歡他們依照每個用戶的使用習慣去推薦該用戶歌曲，以及年末時的統計數據，在人力銀行查了後端工程師的職缺，除了福利吸引人外，也有Code review或各種程式課程。工作內容像是內外部與API和後端的設計開發、故障排除、CI/CD 環境維護和自動化...等，雖然職缺裡需要具備的技能目前還沒學習到，但會以此為目標，繼續朝後端的方向努力。

    描述一件你曾經碰到最困難的事情 如何克服？

    大學前的讀的是高職商科，後來大學跨考到護理系，發現班上同學很多是來自普通高中，因為高職跟高中的學科不同，大一時生物及化學的分數比重佔得多，所以來自高職的我讀得比較辛苦，同學們可以靠高中所學的去準備考試，但我常上課聽不懂，也抓不到重點。會擔心自己背景的關係差其他同學太多，對當時的我來說是大學遇到最困難的事。課業上雖然起跑點輸了別人不少，在調整心態後，跟其他同學組成讀書會，一起複習上課所學並一起討論不懂的地方，參加老師的課後課程，錄音後回去重聽，考試成績也有了起色。勤能補拙能補足自己跟其他人之間的差距，了解如果一件事沒能做好，就是自己不夠努力！

    關於申請網頁，分享開發時的技術心得。

    第一次自己從零開始寫出完整的網頁，發現自己整體架構想的不夠完善就開始做網頁，在過程中臨時想做什麼就加什麼，導致程式碼不夠乾淨，也會做到一半不喜歡呈現的感覺，或是版面跑版，前前後後重做了三次。對我來說最困難的部分是整體排版，像是左右隔開的區塊，應該要設定多少，才不會跑版，一開始對大小寬度數值概念沒那麼好，很多地方慢慢調出來後才漸漸有雛型。也會想把學到的功能都做出來，但畫面會顯的雜亂，因為希望自己的網頁是讓人覺得舒服乾淨，為了放各種功能而導致畫面混亂是本末倒置的，於是捨棄了各種奇怪的特效。即使如此，看著想做的功能可以用程式碼具現化出來，版面排版上或小地方的呈現，應用之前所學及看了網路教學文章做出來後，帶給我很大的成就感。

    其他想要對我們說的事情？-

    一開始就是從彭彭老師的教學開始學習的，很喜歡老師深入淺出的教學，讓新手容易入門。剛好看到Bootcamp的招生訊息，很認同老師的教學理念，也想給自己一個跳脫舒適圈的機會，抱著絕對要成功參加並完成半年的訓練的心情報名的。最後也謝謝老師提供這樣的平台提供大家學習，會持續往成為工程師的方向前進的。
